# Zenodo Data Packaging Pipeline

This directory contains two scripts that together package raw recording data into
per-animal archives for upload to Zenodo.

## Overview

```
simplexAnimalEphy.config  +  zenodo_recordings_cache.csv
              ↓
       packageZenodoData.py        (step 1 — one-time manifest generation)
              ↓
         zenodo_package.py         (step 2 — run on the data server)
              ↓
    <animal>.tar.gz  ×11           (output — ready for Zenodo upload)
```

---

## Step 1 — Generate the manifest and packaging script

**Script:** `packageZenodoData.py`

Run once (or whenever the trial list changes) to produce the packaging script.

```bash
python packageZenodoData.py [--dest /path/to/output] [--dry-run]
```

**What it does:**
1. Reads `../simplexAnimalEphy.config` (YAML) to find which trial numbers belong to
   each animal and recording day.
2. Reads `zenodo_recordings_cache.csv` to map recording dates to the actual session
   folder names on disk (e.g. `2022.04.26_000`).
3. Resolves the full path to every trial folder and checks that it exists on the server.
4. Writes `zenodo_manifest.txt` — a human-readable list of all included trial folders.
5. Writes `zenodo_package.py` — a self-contained script with the manifest baked in,
   ready to run on the data server.

Use `--dry-run` to check paths and write the manifest without overwriting
`zenodo_package.py`.

**Inputs:**
| File | Purpose |
|------|---------|
| `../simplexAnimalEphy.config` | Trial numbers per animal and recording day |
| `zenodo_recordings_cache.csv` | Maps recording dates to session folder names on disk |

**Outputs:**
| File | Purpose |
|------|---------|
| `zenodo_manifest.txt` | Human-readable list of all trial folders included |
| `zenodo_package.py` | Self-contained packaging script (see Step 2) |

---

## Step 2 — Package the raw data

**Script:** `zenodo_package.py`

Run on the machine that holds the raw data to create the Zenodo archives.

```bash
python zenodo_package.py [--dest /path/to/output] [--quality 85] [--animal 220211_f38]
```

**What it does:**
For each animal in the manifest:
1. Iterates over all session/trial folders listed in `MANIFEST`.
2. Copies all `.ma` files (HDF5, recorded by ACQ4) as-is.
3. Converts each behavioral video (`CameraGigEBehavior/video_000.ma`) from HDF5 to
   a standard MJPG `.avi` file for broad compatibility.
4. Creates a single `<animal>.tar.gz` archive from the staged data.
5. Removes the staging directory after archiving.
6. Skips an animal if its archive already exists (safe to resume after interruption).

When run without `--animal` (full packaging), also copies `README_zenodo.md` as
`README.md` into the output directory alongside the archives.

**Options:**
| Option | Default | Description |
|--------|---------|-------------|
| `--dest` | `/media/HDnyc_data/.../packagedData` | Output directory for `.tar.gz` archives |
| `--quality` | `85` | MJPG compression quality (0–100) |
| `--animal` | *(all)* | Process a single animal only |

**Source data path:** `/media/invivodata2/altair_data/dataMichael`

## Output structure

Each archive expands to:

```
<ANIMAL_ID>/
  <YYYY.MM.DD_NNN>/
    locomotionEphys2Motor60sec_TTT/
      AxoPatch200_2.ma
      DaqDevice.ma
      RotaryEncoder.ma
      CameraGigEBehavior/
        video_000.avi       ← converted from video_000.ma
        daqResult.ma
```

See `README_zenodo.md` for a full description of the data format and contents.

---

## When to re-run Step 1

`zenodo_package.py` is generated by `packageZenodoData.py` and has the manifest
baked in. Re-run Step 1 only if:
- The trial list in `simplexAnimalEphy.config` changes.
- New animals are added.
- The destination path changes.

Otherwise, `zenodo_package.py` can be used directly without re-running Step 1.
